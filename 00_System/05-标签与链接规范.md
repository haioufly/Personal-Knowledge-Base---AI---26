# 标签与链接规范

## 标签规则

标签统一使用英文 `kebab-case`，用于聚类和机器筛选。

推荐三类标签：

- 领域标签，如 `ai`、`business`、`design`
- 主题标签，如 `knowledge-os`、`network-effect`
- 用途标签，如 `project-reference`、`writing-source`

## 标签使用原则

- 一个文件只保留最关键的 2 到 5 个标签。
- 标签用于筛选，不用于表达完整关系。
- 不混用中文标签和英文标签。

## 双链规则

- 使用 `[[文件名]]` 进行显式连接。
- 每个知识节点至少保留 2 个相关链接，目标 3 个。
- Thinking 文件应链接多个 Knowledge 节点。
- Creation 文件必须能回溯到知识或思考来源。

## 连接优先级

1. Knowledge -> Knowledge
2. Thinking -> Knowledge
3. Project -> Knowledge / Thinking
4. Creation -> Knowledge / Thinking

## 连接判断标准

出现以下情况之一，就应建立双链：

- 两个知识存在直接因果或解释关系
- 一个框架需要另一个概念作为组成部分
- 一个创作观点依赖某个知识节点支撑
