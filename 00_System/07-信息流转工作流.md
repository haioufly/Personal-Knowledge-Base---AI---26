# 信息流转工作流

## 标准路径

`01_Input -> 02_Knowledge -> 03_Thinking -> 04_Project -> 05_Creation -> 06_Assets -> 90_Archive`

## 处理规则

### 每日

- 把新文章、书摘、视频记录、对话和灵感放入 `01_Input`。

### 每周

- 从 Input 中挑选值得长期沉淀的内容。
- 把可以明确命名、解释和复用的内容提炼为 `02_Knowledge`。

### 每月

- 归并知识节点。
- 补齐知识之间的链接。
- 把多知识组合抽象为 `03_Thinking`。

### 按项目推进

- 在 `04_Project` 中引用知识和思考节点。
- 避免在项目文档中重复抄写基础定义。

### 输出阶段

- 在 `05_Creation` 中完成文章、方案和研究草稿。
- 成熟后把可长期经营的成果迁移到 `06_Assets`。

## 判断标准

- 不能稳定复用: 留在 Input。
- 能被稳定引用: 进入 Knowledge。
- 组合多个知识解决问题: 进入 Thinking。
- 针对具体目标展开执行: 进入 Project。
- 面向对外输出: 进入 Creation。
- 进入长期经营周期: 进入 Assets。
