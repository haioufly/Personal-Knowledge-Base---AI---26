# 元数据规范

## 目标

frontmatter 是知识库的稳定机器接口，用于 AI Agent、脚本和检索工具读取文件属性。

## 知识层强制字段

```yaml
---
id: KNOW-AI-001
type: knowledge
domain: AI
tags:
  - knowledge-os
  - personal-kb
status: active
source_type: synthesis
created: 2026-03-06
updated: 2026-03-06
---
```

字段说明：

- `id`: 唯一标识，仅知识层强制。
- `type`: 文件类型，如 `knowledge`、`thinking`、`project`、`creation`、`asset`、`input`。
- `domain`: 一级领域。
- `tags`: 英文 `kebab-case` 标签数组。
- `status`: `active / draft / archived`。
- `source_type`: `book / paper / article / video / conversation / synthesis`。
- `created`: 创建日期。
- `updated`: 最近更新日期。

## 其他层推荐字段

```yaml
---
type: thinking
tags:
  - framework
created: 2026-03-06
updated: 2026-03-06
---
```

## 使用规则

- `created` 和 `updated` 使用 `YYYY-MM-DD`。
- 标签不要写成自然语言句子。
- `domain` 与目录尽量保持一致。
- 不在正文中重复 frontmatter 中已经表达的机器字段。

## 自动校验

可以通过脚本执行知识层元数据校验：

```bash
python3 scripts/kb_tools.py validate
```

当前会检查：

- 必填字段是否缺失
- `id` 是否重复
- `type` 是否为 `knowledge`
- `domain` 是否和文件名、目录一致
- `status`、`source_type`、`created`、`updated`、`tags` 是否符合规则
